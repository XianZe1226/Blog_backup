# 建设过程中遇到的问题及解决

---

## 一、Node.js 环境问题

> **问题描述**：终端显示没有 node 版本被激活，导致 hexo 命令无法执行。

### 解决方法

**1. 激活 nvm**

在 zsh 里运行：
```bash
source ~/.nvm/nvm.sh
```

**2. 查看已安装的 Node 版本**
```bash
nvm list
```

**3. 选择一个 Node 版本**
```bash
nvm use 20
```

**4. 确认版本**
```bash
node -v
npm -v
```

## 二、文章封面默认设置未生效

> **问题描述**：在 `_config.butterfly.yml` 中设置的 `default_top_img` 路径没有生效。

### 解决方法

**错误写法**：
```yaml
default_top_img: /source/images/top.jpg
```

**正确写法**：
```yaml
default_top_img: /images/top.jpg
```

**原因**：`/images/1.jpg` 会指向 `source/images/1.jpg`，这是 Hexo 的约定，生成时会自动复制到 `public/images/1.jpg`。

> **注意**：YAML 语法要求冒号后必须有空格。

## 三、Hexo 部署问题（2026-06-30 新增）

> **问题描述**：修改了文章日期后，执行 `hexo deploy` 部署，但网站内容没有更新，文章页面出现 404 错误。

### 问题分析

1. **目录混淆**：有两个相关目录
   - `~/Blog` — Hexo 源码仓库（对应 `Blog_backup.git`）
   - `~/Blogs/Blog1` — GitHub Pages 仓库（对应 `XianZe1226.github.io.git`）

2. **缓存问题**：之前没有清除 Hexo 缓存就直接部署，导致旧内容没有被更新。

3. **静态文件问题**：源码仓库中包含了 Hexo 生成的静态文件（`index.html`、`archives/` 等），这些文件不应该在源码仓库中。

### 解决方法

**正确的 Hexo 部署三连**：
```bash
hexo clean && hexo generate && hexo deploy
```

- `hexo clean` — 清除缓存和 `public/` 目录
- `hexo generate` — 重新生成静态文件
- `hexo deploy` — 部署到 GitHub Pages

**更新 `.gitignore`**，添加以下内容排除生成的静态文件：
```gitignore
# Hexo generated files
index.html
archives/
categories/
tags/
css/
js/
images/
img/
link/
shuoshuo/
photo/
photogroup/
search.xml
404.html
```

### 部署流程总结

```
~/Blog (源码)
    ↓ 修改文章
hexo clean
    ↓ 清除缓存
hexo generate
    ↓ 生成静态文件
hexo deploy
    ↓ 推送到 XianZe1226.github.io.git
GitHub Pages 自动更新
```

> **注意**：修改内容后一定要执行完整的三连命令，否则可能出现缓存不更新或 404 错误。
