# 建设过程中遇到的问题及解决

---

## 一、Node.js 环境问题

> **问题描述**：终端显示没有 node 版本被激活，导致 hexo 命令无法执行。

### 解决方法

**1. 激活 nvm**

在 zsh 里运行：
```bash
source ~/.nvm/nvm.sh
```

**2. 查看已安装的 Node 版本**
```bash
nvm list
```

**3. 选择一个 Node 版本**
```bash
nvm use 20
```

**4. 确认版本**
```bash
node -v
npm -v
```

---

## 二、文章封面默认设置未生效

> **问题描述**：在 `_config.butterfly.yml` 中设置的 `default_top_img` 路径没有生效。

### 解决方法

**错误写法**：
```yaml
default_top_img: /source/images/top.jpg
```

**正确写法**：
```yaml
default_top_img: /images/top.jpg
```

**原因**：`/images/1.jpg` 会指向 `source/images/1.jpg`，这是 Hexo 的约定，生成时会自动复制到 `public/images/1.jpg`。

> **注意**：YAML 语法要求冒号后必须有空格。

## 三、Hexo 手动部署问题

> **问题描述**：修改了文章日期后，执行 `hexo deploy` 部署，但网站内容没有更新，文章页面出现 404 错误。

### 问题分析

1. **目录混淆**：有两个相关目录
   - `~/BlogProject/Blog_backup` — Hexo 源码仓库（对应 `Blog_backup.git`）
   - `~/BlogProject/Blog_pages` — GitHub Pages 仓库（对应 `XianZe1226.github.io.git`）

2. **缓存问题**：之前没有清除 Hexo 缓存就直接部署，导致旧内容没有被更新。

### 解决方法

**正确的 Hexo 部署三连**：
```bash
hexo clean && hexo generate && hexo deploy
```

- `hexo clean` — 清除缓存和 `public/` 目录
- `hexo generate` — 重新生成静态文件
- `hexo deploy` — 部署到 GitHub Pages

> **注意**：修改内容后一定要执行完整的三连命令，否则可能出现缓存不更新或 404 错误。

## 四、GitHub Actions 自动部署问题

> **问题描述**：配置了 GitHub Actions 自动部署到 GitHub Pages，但部署后网站布局严重错乱，CSS 样式和 JS 脚本全部丢失。

### 问题分析

1. **`.gitignore` 误排除**：在 `.gitignore` 中添加了 `css/`、`js/` 等规则来排除源码仓库中的生成文件，但这些规则同时影响了 GitHub Actions 的部署流程，导致 `public/` 目录中的 CSS/JS 文件没有被推送到 `gh-pages` 分支。

2. **主题源文件未提交**：Butterfly 主题的 `themes/butterfly/source/` 目录（包含 `.styl` 样式源文件和 `.js` 脚本文件）没有被 git 跟踪。GitHub Actions 在远端构建时找不到这些文件，导致 Stylus 无法编译成 CSS，JS 文件也无法被复制。

3. **路径问题**：`_config.butterfly.yml` 中 `inject.head` 里的 `custom.css` 路径是硬编码的 `/css/custom.css`，部署到子路径（`/Blog_backup/`）后需要改为 `/Blog_backup/css/custom.css`。

### 解决方法

**1. 精简 `.gitignore`**，只保留必要的排除项：
```gitignore
.DS_Store
Thumbs.db
db.json
*.log
node_modules/
public/
.deploy*/
_multiconfig.yml
```

> 不要添加 `css/`、`js/`、`images/` 等规则，否则会影响 GitHub Actions 部署。

**2. 确保主题源文件已提交**：
```bash
git add themes/butterfly/source/
git commit -m "添加主题源文件"
```

**3. 安装主题依赖**（`package.json` 中需包含）：
```bash
npm install hexo-renderer-stylus hexo-renderer-pug --save
```

**4. 配置 GitHub Actions workflow**（`.github/workflows/deploy.yml`）：
```yaml
name: Deploy Hexo to GitHub Pages

on:
  push:
    branches:
      - main

permissions:
  contents: write

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout source
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm install

      - name: Clean and Generate
        run: npx hexo clean && npx hexo generate

      - name: Deploy to gh-pages
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          branch: gh-pages
          folder: public
          clean: true
```

**5. 仓库设置**：在 GitHub 仓库的 Settings → Pages → Source 中选择 `gh-pages` 分支。

### 部署流程总结

```
本地修改文章
    ↓
git add / git commit / git push
    ↓
GitHub Actions 自动触发
    ↓
npm install（安装依赖）
    ↓
hexo clean && hexo generate（生成静态文件）
    ↓
JamesIves action 部署到 gh-pages 分支
    ↓
GitHub Pages 自动更新网站
```

> **注意**：`.gitignore` 中的规则会影响 GitHub Actions 的部署，不要添加排除生成文件的规则。主题的 `source/` 目录必须提交到 git，否则远端构建时 CSS/JS 无法生成。
