# 建设过程中遇到的问题及解决

---

## 一、Node.js 环境问题

> **问题**：终端没有激活 node 版本，hexo 命令无法执行。

```bash
source ~/.nvm/nvm.sh   # 激活 nvm
nvm list                # 查看已安装版本
nvm use 20              # 选择版本
node -v && npm -v       # 确认
```

---

## 二、文章封面默认设置未生效

> **问题**：`_config.butterfly.yml` 中 `default_top_img` 路径没有生效。

```yaml
# 错误
default_top_img: /source/images/top.jpg

# 正确（Hexo 约定：/images/ 自动指向 source/images/）
default_top_img: /images/top.jpg
```

---

## 三、双轨部署策略

博客维护两套独立的仓库和部署链路，互不干扰：

| | 正式站 | 备份站 |
|---|---|---|
| **本地目录** | `~/BlogProject/Blog_pages` | `~/BlogProject/Blog_backup` |
| **GitHub 仓库** | `XianZe1226/XianZe1226.github.io` | `XianZe1226/Blog_backup` |
| **网站地址** | `xianze1226.github.io` | `xianze1226.github.io/Blog_backup/` |
| **部署方式** | `hexo deploy` 手动推送 | `git push` → GitHub Actions 自动部署 |
| **Pages 分支** | `main` | `gh-pages` |

### 正式站操作（Blog_pages）

改内容 → 在 `Blog_backup` 目录编辑 Markdown → 复制到 `Blog_pages` → 执行：

```bash
cd ~/BlogProject/Blog_pages
hexo clean && hexo generate && hexo deploy
```

### 备份站操作（Blog_backup）

改内容 → 在 `Blog_backup` 目录编辑 Markdown → 执行：

```bash
cd ~/BlogProject/Blog_backup
git add . && git commit -m "更新内容" && git push origin main
```

push 后 GitHub Actions 自动构建部署，无需手动操作。

## 四、手动部署：内容不更新 / 404

> **问题**：执行 `hexo deploy` 后网站没变化或文章 404。

**原因**：没清缓存，旧内容残留。

**解决**：每次部署必须三连：
```bash
hexo clean && hexo generate && hexo deploy
```

---

## 五、GitHub Actions 部署：CSS/JS 全部丢失，页面布局错乱

> **问题**：Actions 部署成功，但网站没有样式、没有交互，布局完全错乱。

排查后发现三个原因：

### 原因 1：`.gitignore` 误排除

之前在 `.gitignore` 中加了 `css/`、`js/` 等规则来排除源码仓库中的生成文件，但这些规则同时影响了 Actions 的部署流程。

**解决**：`.gitignore` 只保留基础规则，不要加排除生成文件的规则：
```gitignore
.DS_Store
Thumbs.db
db.json
*.log
node_modules/
public/
.deploy*/
_multiconfig.yml
```

### 原因 2：主题源文件未提交

Butterfly 主题的 `themes/butterfly/source/` 目录（`.styl` 样式源文件和 `.js` 脚本文件）没有被 git 跟踪。Actions 在远端构建时找不到这些文件，Stylus 无法编译，JS 也无法复制。

**解决**：确保主题源文件已提交：
```bash
git add themes/butterfly/source/
git commit -m "添加主题源文件"
git push
```

### 原因 3：主题渲染器依赖缺失

`hexo-renderer-stylus` 和 `hexo-renderer-pug` 不在根目录 `package.json` 中，Actions 执行 `npm install` 时不会安装它们。

**解决**：
```bash
npm install hexo-renderer-stylus hexo-renderer-pug --save
```

### 完整 Actions workflow

文件路径：`.github/workflows/deploy.yml`

```yaml
name: Deploy Hexo to GitHub Pages

on:
  push:
    branches:
      - main

permissions:
  contents: write

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout source
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm install

      - name: Clean and Generate
        run: npx hexo clean && npx hexo generate

      - name: Deploy to gh-pages
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          branch: gh-pages
          folder: public
          clean: true
```

### GitHub 仓库设置

Settings → Pages → Source → 选择 `gh-pages` 分支，文件夹选 `/ (root)`。

> **总结**：Actions 部署出问题，99% 是文件没提交全或 `.gitignore` 排多了。检查 `git ls-tree HEAD themes/butterfly/source/` 是否有输出，检查 `.gitignore` 是否有 `css/`、`js/` 等排除规则。
